# Dongle SMS Manager — Build Specification (LOCAL)

> **App name:** `dongle-php`  
> **Deployment:** Runs on the FreePBX server itself. All connections are localhost.  
> **Driver:** chan_dongle ONLY — Huawei GSM USB modems on Asterisk.  
> **Multi-dongle:** Supports 1 to unlimited USB modems simultaneously.  
> **No SIP, no PJSIP, no trunks, no extensions, no IVR, no general PBX features.**

---

## System Context

chan_dongle is an Asterisk channel driver for Huawei USB GSM/3G modems. Each physical USB modem plugged into the server appears as a separate dongle device in Asterisk (dongle0, dongle1, dongle2, etc). Each dongle has its own SIM card, phone number, operator, and signal level. They operate independently.

This app manages ALL connected dongles from a single interface. Every feature — SMS, USSD, monitoring, reports — works per-dongle and across all dongles.

### Architecture

```
FreePBX Server (everything on one machine)
│
├── USB Hub ──▶ Huawei Modem 1 ──▶ /dev/ttyUSB0, /dev/ttyUSB1
│             ▶ Huawei Modem 2 ──▶ /dev/ttyUSB2, /dev/ttyUSB3
│             ▶ Huawei Modem 3 ──▶ /dev/ttyUSB4, /dev/ttyUSB5
│             ▶ ... (unlimited)
│
├── Asterisk + chan_dongle
│   ├── dongle0 ──▶ Modem 1 (SIM: Operator A, +1234567890)
│   ├── dongle1 ──▶ Modem 2 (SIM: Operator B, +0987654321)
│   ├── dongle2 ──▶ Modem 3 (SIM: Operator A, +1122334455)
│   └── ... (matches USB modems)
│
├── Apache ──▶ /var/www/html/dongle-php/
│                    │
│                    ├── AMI (127.0.0.1:5038) ──▶ chan_dongle commands
│                    └── MariaDB (localhost) ──▶ dongle_manager DB
│
└── Config:
    ├── /etc/freepbx.conf          (DB + AMI credentials)
    ├── /etc/asterisk/dongle.conf  (dongle device definitions)
    └── /etc/asterisk/manager.conf (AMI users)
```

- App URL: `http://<server-ip>/dongle-php/`
- No firewall changes needed
- No remote database grants needed

## Credential Auto-Detection

The installer (install.php) reads credentials automatically from FreePBX. Nothing is hardcoded.

### Database Credentials

Check in order, use the first found:

**Priority 1:** `/etc/freepbx.conf` (PHP file)
```php
$amp_conf['AMPDBHOST'] = 'localhost';
$amp_conf['AMPDBUSER'] = 'freepbxuser';
$amp_conf['AMPDBPASS'] = 'some_password';
```
Read by including the file and accessing the `$amp_conf` array.

**Priority 2:** `/etc/amportal.conf` (flat key=value, older FreePBX)
```
AMPDBHOST=localhost
AMPDBUSER=freepbxuser
AMPDBPASS=some_password
```
Read by parsing each line, splitting on `=`.

The installer:
1. Reads FreePBX DB credentials from whichever file exists
2. Connects to MariaDB using those credentials
3. Creates database `dongle_manager` (separate from FreePBX's `asterisk` DB)
4. Creates all tables
5. Saves working credentials to `config.php`

### AMI Credentials

From the same config files:

**From `/etc/freepbx.conf`:**
```php
$amp_conf['AMPMGRUSER'] = 'admin';
$amp_conf['AMPMGRPASS'] = 'amp111';
```

**From `/etc/amportal.conf`:**
```
AMPMGRUSER=admin
AMPMGRPASS=amp111
```

AMI host is always `127.0.0.1`, port always `5038`.

## chan_dongle AMI Commands Reference

These are the ONLY AMI commands this app uses. All are chan_dongle specific.

### Query Commands (read data)

**List all dongles with full status:**
```
Action: DongleShowDevices
```
Returns one `DongleDeviceEntry` event per dongle with fields:
- `Device` — dongle name (dongle0, dongle1, ...)
- `AudioSetting` — audio device path
- `DataSetting` — data device path
- `IMEISetting` — IMEI number
- `IMSISetting` — IMSI number
- `NumberSetting` — phone number
- `State` — Free, Busy, Ring, Init, Error
- `CallWaiting` — Disabled/Enabled
- `CurrentDeviceState` — Start, Stop, Restart
- `SubscriberNumber` — phone number from SIM
- `ContentType` — UTF-8 etc
- `RSSI` — signal strength (0-31 scale, 99=unknown)
- `ProviderName` — operator name
- `Manufacturer` — modem manufacturer
- `Model` — modem model
- `Firmware` — firmware version
- `IMEI` — actual IMEI
- `IMSI` — actual IMSI
- `GSMRegistrationStatus` — Registered, HomeNetwork, Searching, Denied, Unknown, Roaming
- `IncomingSMS` — count
- `OutgoingSMS` — count
- `IncomingCall` — count
- `OutgoingCall` — count
- `TasksInQueue` — pending tasks

**RSSI to percentage conversion:**
```
RSSI 0     = -113 dBm or less  =  0%
RSSI 1     = -111 dBm          =  3%
RSSI 2-30  = -109 to -53 dBm  = linear 6% to 97%
RSSI 31    = -51 dBm or more   = 100%
RSSI 99    = unknown/no signal = 0%
Formula: percentage = round((rssi / 31) * 100) for rssi 0-31
```

### Action Commands (change state)

**Send SMS (specify which dongle):**
```
Action: DongleSendSMS
Device: dongle0
Number: +1234567890
Message: Hello from dongle0
```

**Send USSD (specify which dongle):**
```
Action: DongleSendUSSD
Device: dongle0
USSD: *100#
```

**Restart a specific dongle:**
```
Action: DongleRestart
Device: dongle0
When: now
```

**Stop a specific dongle:**
```
Action: DongleStop
Device: dongle0
When: now
```

**Start a specific dongle:**
```
Action: DongleStart
Device: dongle0
```

**Reset a specific dongle (full hardware reset):**
```
Action: DongleReset
Device: dongle0
```

### AMI Events (incoming, captured by worker)

**Incoming SMS received:**
```
Event: DongleNewSMS
Device: dongle0
From: +1234567890
Message: Hello
Received: 2024-01-15 10:30:00
```

**USSD response received:**
```
Event: DongleNewUSSD

```

**Dongle status change:**
```

```

```bash

# Show one dongle detail

# Send SMS via CLI
```

## Database Schema

Database name: `dongle_manager`

All tables track which dongle is involved. Every query can be filtered by dongle.

### Table: `dongles`

One row per physical dongle. Updated by the background worker from AMI data.

```sql
CREATE TABLE dongles (
    id              INT AUTO_INCREMENT PRIMARY KEY,
    device          VARCHAR(50) NOT NULL UNIQUE,   -- dongle0, dongle1, ...
    imei            VARCHAR(20) DEFAULT '',
    imsi            VARCHAR(20) DEFAULT '',
    phone_number    VARCHAR(30) DEFAULT '',         -- from SIM
    operator        VARCHAR(100) DEFAULT '',        -- provider name
    manufacturer    VARCHAR(100) DEFAULT '',
    model           VARCHAR(100) DEFAULT '',
    firmware        VARCHAR(100) DEFAULT '',
    signal_rssi     TINYINT DEFAULT 0,              -- 0-31, 99=unknown
    signal_percent  TINYINT DEFAULT 0,              -- calculated 0-100
    gsm_status      VARCHAR(50) DEFAULT '',         -- Registered, Searching, etc
    state           VARCHAR(30) DEFAULT '',          -- Free, Busy, Error, etc
    enabled         TINYINT(1) DEFAULT 1,           -- enabled in this app
    sms_in_count    INT DEFAULT 0,                  -- lifetime incoming SMS
    sms_out_count   INT DEFAULT 0,                  -- lifetime outgoing SMS
    tasks_in_queue  INT DEFAULT 0,
    last_seen       DATETIME NULL,                  -- last time worker saw it active
    created_at      DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at      DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### Table: `sms_inbox`

Every incoming SMS. Linked to the dongle that received it.

```sql
CREATE TABLE sms_inbox (
    id          INT AUTO_INCREMENT PRIMARY KEY,
    dongle      VARCHAR(50) NOT NULL,               -- which dongle received it
    sender      VARCHAR(30) NOT NULL,               -- sender phone number
    message     TEXT NOT NULL,
    received_at DATETIME NOT NULL,                  -- when SMS arrived at modem
    is_read     TINYINT(1) DEFAULT 0,
    created_at  DATETIME DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_dongle (dongle),
    INDEX idx_sender (sender),
    INDEX idx_received (received_at),
    INDEX idx_read (is_read)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### Table: `sms_outbox`

Every outgoing SMS. Linked to the dongle that sent it.

```sql
CREATE TABLE sms_outbox (
    id          INT AUTO_INCREMENT PRIMARY KEY,
    dongle      VARCHAR(50) NOT NULL,               -- which dongle sent it
    destination VARCHAR(30) NOT NULL,               -- recipient phone number
    message     TEXT NOT NULL,
    status      ENUM('queued','sending','sent','failed') DEFAULT 'queued',
    error       TEXT NULL,                          -- error message if failed
    retry_count TINYINT DEFAULT 0,                  -- how many retries attempted
    sent_at     DATETIME NULL,                      -- when actually sent
    created_at  DATETIME DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_dongle (dongle),
    INDEX idx_status (status),
    INDEX idx_dest (destination),
    INDEX idx_created (created_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### Table: `ussd_history`

Every USSD command sent and response received. Per dongle.

```sql
CREATE TABLE ussd_history (
    id          INT AUTO_INCREMENT PRIMARY KEY,
    dongle      VARCHAR(50) NOT NULL,               -- which dongle was used
    command     VARCHAR(255) NOT NULL,               -- USSD code, e.g. *100#
    response    TEXT NULL,                           -- response from network
    status      ENUM('sent','received','timeout','failed') DEFAULT 'sent',
    created_at  DATETIME DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_dongle (dongle),
    INDEX idx_created (created_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### Table: `system_logs`

Application and dongle event logs.

```sql
CREATE TABLE system_logs (
    id          INT AUTO_INCREMENT PRIMARY KEY,
    level       ENUM('info','warning','error') DEFAULT 'info',
    category    VARCHAR(50) NOT NULL,               -- sms, ussd, dongle, system, worker
    dongle      VARCHAR(50) NULL,                   -- which dongle (NULL for system-wide)
    message     TEXT NOT NULL,
    created_at  DATETIME DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_level (level),
    INDEX idx_category (category),
    INDEX idx_dongle (dongle),
    INDEX idx_created (created_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### Table: `users`

Login accounts.

```sql
CREATE TABLE users (
    id          INT AUTO_INCREMENT PRIMARY KEY,
    username    VARCHAR(50) NOT NULL UNIQUE,
    password    VARCHAR(255) NOT NULL,              -- bcrypt hash
    role        ENUM('admin','operator','viewer') DEFAULT 'viewer',
    last_login  DATETIME NULL,
    created_at  DATETIME DEFAULT CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

**API:**
```
POST api/ussd.php?action=send     body: dongle=dongle0&command=*100#
GET  api/ussd.php?action=history  &page=1&dongle=dongle0
GET  api/ussd.php?action=check    &id=5  (poll for response)
```

**USSD flow:**
1. User submits USSD command
2. API sends `DongleSendUSSD` via AMI, inserts record in `ussd_history` with status=`sent`
3. Frontend polls `api/ussd.php?action=check&id=X` every 2 seconds
4. Worker captures `DongleNewUSSD` event, updates record with response and status=`received`
5. Frontend shows response when status changes
6. If no response after 30 seconds, mark as `timeout`

## Background Worker — cron/worker.php

Runs every minute via cron:
```
* * * * * php /var/www/html/dongle-php/cron/worker.php >> /var/log/dongle-worker.log 2>&1
```

### Worker Steps (every run):

1. **Lock check** — use PID file `/tmp/dongle-worker.pid`. If another instance running, exit.

2. **Connect to AMI** on 127.0.0.1:5038

3. **Query all dongles** — send `DongleShowDevices`, parse each `DongleDeviceEntry` event:
   - If dongle exists in `dongles` table: UPDATE signal, state, operator, gsm_status, counters, last_seen
   - If dongle is new (not in table): INSERT new row
   - If dongle was in table but NOT returned by AMI: mark state as offline (leave last_seen unchanged)

4. **Process outbox queue** — SELECT from `sms_outbox` WHERE status=`queued` ORDER BY created_at LIMIT 5:
   - For each: send `DongleSendSMS` via AMI using the specified dongle
   - Update status to `sending`
   - If AMI returns error immediately, update to `failed` with error message

5. **Listen for events** — read AMI socket for 5 seconds, capture:
   - `DongleNewSMS` → INSERT into `sms_inbox` (dongle, sender=From, message=Message)
   - `DongleNewUSSD` → UPDATE matching `ussd_history` record with response, status=`received`
   - `DongleStatus` → UPDATE dongle state in `dongles` table

6. **Timeout check** — UPDATE `sms_outbox` SET status=`failed`, error=`Send timeout` WHERE status=`sending` AND created_at < NOW() - 120 seconds

7. **USSD timeout** — UPDATE `ussd_history` SET status=`timeout` WHERE status=`sent` AND created_at < NOW() - 30 seconds

8. **Auto-restart frozen dongles** — if a dongle state=`Error` or has not been seen for 5+ minutes but is in `dongle.conf`, send `DongleRestart`

9. **Log cleanup** — DELETE FROM `system_logs` WHERE created_at < NOW() - 90 days

10. **Release lock** — remove PID file

## Installer — install.php

Runs once on first access. Shows a step-by-step wizard.

### Step 1: System Check
- Check PHP version >= 7.4
- Check PHP extensions: pdo_mysql, sockets, json, mbstring
- Check Apache is running
- Check Asterisk is running
- Show result: all green = continue, any red = show fix instructions

### Step 2: Detect FreePBX Credentials
- Read `/etc/freepbx.conf` or `/etc/amportal.conf`
- Display detected: DB host, DB user, DB pass (masked), AMI user, AMI pass (masked)
- Test DB connection — show green/red
- Test AMI connection — show green/red
- If auto-detect fails: show manual input form for DB host, user, pass, AMI user, pass

### Step 3: Create Database
- CREATE DATABASE `dongle_manager` if not exists
- Run all CREATE TABLE statements
- Show table creation results

### Step 4: Detect Dongles
- Send `DongleShowDevices` via AMI
- Show list of detected dongles with status
- Insert into `dongles` table
- If no dongles found: show warning but allow continue

### Step 5: Create Admin Account
- Username input (default: `admin`)
- Password input + confirm
- Create user with role=admin, bcrypt hashed password

### Step 6: Write Config
- Generate `config.php` with detected/entered credentials
- Write lock file `.installed` to prevent re-running
- Show success message
- Redirect to login page

### After Install
- `install.php` checks for `.installed` file. If exists, show "Already installed" and link to app.
- To re-run installer: delete `.installed` file

## Security

- All user input: `htmlspecialchars()` for output, PDO prepared statements for queries
- CSRF token on all POST forms
- Session regeneration on login
- Rate limit SMS sending: max 10 per minute per dongle (configurable)
- API returns JSON even on error (never raw PHP errors)
- AMI credentials never exposed to frontend
- `config.php` should be readable only by www-data: `chmod 640`

---

## Error Handling

- AMI connection failure: log to `system_logs`, show user-friendly message on UI
- Database errors: catch PDOException, log, show generic error
- Worker errors: log to `system_logs` + stdout (for cron log)
- All API responses: always valid JSON `{"success": bool, "message": string, "data": mixed}`
- If a dongle is offline and user tries to send SMS/USSD through it: return clear error message

## Build Notes

- **Zero dependencies** — no composer, no npm, no build step
- Copy folder to `/var/www/html/dongle-php/`, open in browser, run installer
- All external libraries stored locally in `assets/vendor/` — no CDN, no internet required
- Download URLs for vendor files:
  - Tailwind: `https://cdn.tailwindcss.com/3.4.17` → save as `assets/vendor/tailwindcss.js`
  - Chart.js: `https://cdn.jsdelivr.net/npm/chart.js@4/dist/chart.umd.min.js` → save as `assets/vendor/chart.umd.min.js`
  - Font Awesome: download from `https://fontawesome.com/download` → extract `css/all.min.css` and `webfonts/` folder into `assets/vendor/fontawesome/`
- Target: CentOS/AlmaLinux or Debian/Ubuntu with FreePBX + chan_dongle already installed
- PHP extensions required: `pdo_mysql`, `sockets`, `json`, `mbstring`
- The app reads dongle state from Asterisk in real-time — it does NOT modify Asterisk config files
- chan_dongle must already be installed and configured in `/etc/asterisk/dongle.conf`
- Modems must already be recognized by the OS (`/dev/ttyUSB*` devices present)
